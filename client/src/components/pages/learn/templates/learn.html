{{__ 'Starting Wilson'}}

{{__ 'Once Wilson is installed, you can run it using grunt from the commandline. There are two modes for running wilson:'}} {{__ 'Development and Production'}}.

{{__ 'Development Mode'}}

grunt server

Running wilson in development mode will start the server as well as a watcher task for the project. This is a blocking process that will stay open in the console and detect changes to the project. Any new changes will force an on-the-fly rebuild and refresh of the server, allowing you make changes fast and have them immediately available. Please note that in development mode, the server runs on port 3000.

{{__ 'Production Mode'}}

sudo grunt server:dist

When running in production mode, the watch task will not run and therefore the process will not block. The server will be started in the background and the command will complete. In order to stop the server in production mode run the following command:

sudo grunt stopServer

Note: in production mode, the server runs on port 8080.

{{__ 'Components'}}

{{__ 'Components are the core of your Wilson application. They are the pieces that will be used to construct your experiences. Building a Wilson application is essentially building a library of components that can be reused as your application grows. Components are meant to be leveraged and reused by other developers so that functionality is centralized and code is not duplicated.'}}

{{__ 'At their core, components are simply an Angularjs directive that defines a custom controller and specifies itself to be used as an element. What this means is that when we define a component, we are really creating a new type of HTML tag/element that can be used in the markup to instantiate it. Understanding components starts with understanding Anguarljs directives, you can read more'}} here.

{{__ 'In Wilson there are two categories of components: building-blocks and pages. Building-blocks are exactly what they sound like, components used to build pages or other building-block components. Pages are components explicitly used to represent a top-level page and they are the only components that can be used as route targets.'}}

{{__ 'Creating a Component'}}

yo component

Creating a component is as easy as running this command and following the prompts. This process will create all necessary directories and files for a new empty component.
